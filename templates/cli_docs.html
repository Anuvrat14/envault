CLI Integration

Use your vault secrets directly in the terminal
The dotward CLI tool

A lightweight Python script that talks to your running Dotward vault over localhost. Fetch individual secrets, inject an entire project's variables into any process, or write new values — all without ever touching a plaintext .env file on disk.

Download CLI Script Manage Token
Installation
Python 3.8+ is required. The CLI uses only standard library modules — no pip install needed.
1
Download the CLI script

Click the button above or use curl to download directly.

# Using curl
curl -o dotward {{ request.url_root }}cli/download
2
Make it executable and install globally
chmod +x dotward
sudo mv dotward /usr/local/bin/dotward

Or keep it local: ./dotward

3
Generate and save your CLI token

Go to Settings → CLI Integration, click Generate Token, then Download Token File.

# Create the .dotward directory and move the token file
mkdir -p ~/.dotward
mv ~/Downloads/dotward_cli_token ~/.dotward/cli_token
chmod 600 ~/.dotward/cli_token
4
Verify the connection

Make sure Dotward is running and the vault is unlocked, then:

$ dotward status
Dotward 1.1.0 — unlocked
Commands
dotward status
Check whether the vault is reachable and unlocked.
$ dotward status
Dotward 1.1.0 — unlocked
dotward projects
List all projects in the vault with variable counts.
$ dotward projects
My App Production 12 vars Production environment
My App Staging 9 vars
Internal Tools 4 vars
dotward list <project>
List all variable keys in a project. Flagged variables show a risk level tag.
$ dotward list "My App Production"
API_KEY
DATABASE_URL
JWT_SECRET
STRIPE_KEY [high]
WEBHOOK_SECRET
dotward get <project> <KEY>
Decrypt and print a single variable value. Pipe-friendly — outputs just the value.
$ dotward get "My App" DATABASE_URL
postgres://user:pass@localhost:5432/myapp

# Capture in a shell variable
DB=$(dotward get "My App" DATABASE_URL)
dotward set <project> <KEY> [value]
Set or update a variable. Omit the value to be prompted securely (hidden input), or pipe a value from stdin.
# Inline value
$ dotward set "My App" API_KEY sk-abc123
Set API_KEY

# Read from stdin (pipe)
$ cat private.key | dotward set "My App" PRIVATE_KEY

# Prompted (hidden) — good for CI setup
$ dotward set "My App" DB_PASSWORD
Value for DB_PASSWORD: ****
dotward export <project> [--output <file>]
Print all variables as KEY=VALUE lines. Without --output, writes to stdout — useful for piping.
# Print to terminal
$ dotward export "My App"
API_KEY=sk-abc123
DATABASE_URL=postgres://...

# Write to a file
$ dotward export "My App" --output .env
Exported 12 variables → .env
dotward inject <project> -- <command> [args...]
Run any command with the project's variables automatically injected as environment variables. This is the recommended pattern — your code reads from process.env as normal, but the values never touch disk.
# Start a dev server with vault vars
$ dotward inject "My App" -- npm run dev

# Run a Python script
$ dotward inject "My App" -- python manage.py migrate

# Run a Docker build with secrets
$ dotward inject "My App" -- docker compose up
Shell & Project Integration
Replace your .env loader

Instead of committing a .env file or running source .env, prefix any command with dotward inject.

package.json
{ "scripts": { "dev": "dotward inject \"My App\" -- next dev", "build": "dotward inject \"My App\" -- next build", "test": "dotward inject \"My App\" -- jest" } }
Makefile
run:
    dotward inject "My App" -- go run ./cmd/server

migrate:
    dotward inject "My App" -- go run ./cmd/migrate
Shell alias (add to ~/.zshrc or ~/.bashrc)
# Short alias for your main project
alias dev='dotward inject "My App" --'

# Then just:
$ dev npm start
$ dev python app.py
Suppressing False Positives — .dotwardignore

Create a .dotwardignore file in the root of any project to silence findings that you've reviewed and confirmed are safe. The scanner reads it automatically — no flags needed.

Format
.dotwardignore
# Suppress a specific line
app/routes.py:35

# Suppress an entire file
scripts/release.sh

# Comments and blank lines are ignored
Example workflow
# Scan your project — 3 false positives flagged
$ dotward scan --deep ./my-project

# Review them, confirm safe, add to ignore file
$ echo "app/routes.py:35" >> my-project/.dotwardignore
$ echo "app/routes.py:36" >> my-project/.dotwardignore

# Clean scan
$ dotward scan --deep ./my-project
✓ No secrets found in 214 files.
Only suppress findings you've manually reviewed. Adding real secrets to .dotwardignore doesn't make them safe — it just stops the scanner from warning you about them.

Commit .dotwardignore to your repo so the whole team shares the same suppressions. The file itself contains no secrets — only file paths and line numbers.

Configuration
Variable Default Description
DOTWARD_TOKEN ~/.dotward/cli_token Override the CLI token. Useful in CI or Docker.
DOTWARD_URL http://127.0.0.1:5177 Override the server URL if you've changed the port.
Security Notes
The vault must be unlocked for CLI commands to work. If the app is locked (or not running), commands will fail immediately — your secrets are never accessible without the master password.
All API traffic stays on localhost — nothing leaves your machine.
The token file should be chmod 600 (owner read-only). Anyone with the token can read your vault while it's unlocked.
Regenerating the token immediately invalidates the old one. Do this if the token file is ever accidentally exposed.
dotward inject injects variables into a subprocess environment only — they are never written to disk or logged.
Do not commit your token to .env, dotfiles, or any repository. Add ~/.dotward/ to your global .gitignore.
~/.gitignore_global

Troubleshooting
Error Cause & Fix
Cannot connect to Dotward The app isn't running. Open Dotward and unlock the vault first.
Vault is locked App is running but vault is locked. Click Unlock in the app.
Invalid token Wrong token or regenerated since last download. Re-download from Settings.
CLI token not generated Go to Settings → CLI Integration → Generate Token.
No CLI token found Token file missing. Download it from Settings and save to ~/.dotward/cli_token.
Project not found Project name is case-insensitive but must match exactly. Run dotward projects to see exact names.
